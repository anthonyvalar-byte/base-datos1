# Prueba 1 – Base de Datos

## Situación Propuesta

La empresa **TecnoMarket S.A.**, una tienda dedicada a la venta de dispositivos electrónicos (computadoras, teléfonos, accesorios y componentes), está experimentando problemas al manejar su información mediante hojas de cálculo. Actualmente la empresa lleva el registro de clientes, facturas, inventario, proveedores y ventas de forma manual, lo que ha generado múltiples inconvenientes:

* **Pérdida de información** por errores humanos al copiar datos entre archivos.
* **Duplicidad de registros** de clientes y productos.
* **Dificultad para rastrear ventas**, ya que cada vendedor maneja sus propios documentos.
* **Inventario desactualizado**, causando faltantes o exceso de producto.
* **Imposibilidad de generar reportes confiables** para la toma de decisiones.

Para solucionar estos problemas, la empresa requiere implementar una **base de datos relacional** que centralice toda la información y permita automatizar procesos clave.

## Necesidades de la Entidad

La base de datos debe permitir:

1. **Registrar clientes**, incluyendo sus datos personales y método de contacto.
2. **Administrar el catálogo de productos**, con precios, descripciones y existencias.
3. **Controlar el inventario** y actualizarlo automáticamente con cada venta.
4. **Registrar proveedores** y sus productos.
5. **Registrar ventas**, indicando vendedor, cliente, fecha y productos vendidos.
6. **Generar reportes** como ventas por día, productos más vendidos o clientes frecuentes.

## Objetivo del Proyecto

Diseñar una base de datos eficiente que permita:

* Reducir errores.
* Agilizar la consulta de información.
* Facilitar la toma de decisiones basada en datos reales.
* Mejorar la relación con los clientes mediante registros organizados.

Este archivo README.md documenta la situación inicial y servirá como base para el diseño de la solución en entregas posteriores.
